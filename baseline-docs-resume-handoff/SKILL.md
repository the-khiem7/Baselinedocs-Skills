# Baseline Docs Resume Handoff

## Purpose

Create a short handoff so another actor can continue safely.

## Use When

- ownership changes
- another agent will continue implementation
- the user wants a short actionable handoff instead of full baseline reading

## Core Behavior

1. Extract current truth.
2. Summarize open risks and unfinished work.
3. State the next expected action.
4. Highlight pitfalls or assumptions the new owner must know.

## Primary Output

- concise handoff brief

## Non-Goals

- not for baseline initialization
- not for broad codebase syncing
